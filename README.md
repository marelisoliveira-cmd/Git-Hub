Aprendendo a mexer no GitHub: durante a aula, aprendemos a criar um repositório, adicionar um README (que é criado e nunca edita dependências), escolher a visibilidade, adicionar licença (que pode ser exibida, mas não editada; você escolhe se precisa ser pago para visualizar ou se pode visualizar sem editar), e branches, que são subpastas. Também aprendemos sobre commits históricos das atualizações e Codespaces (espaço de trabalho temporário no VS Code).

No commit, deve-se colocar a descrição do que foi feito na edição. Aprendemos a usar o comando git clone com o link no Visual Studio Code (primeiro, vai no Code e copia o link). O aprendizado sobre extensões no Marketplace ficou um pouco vago, assim como o Live Share, que é uma ferramenta de colaboração de código e permite programar em conjunto.

Aprendemos também que é possível dividir a tela no Visual Studio Code, mas seria interessante revisar esse ponto mais uma vez.
